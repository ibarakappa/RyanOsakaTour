二、網頁應用程式產出要求 (請嚴格遵守)

➤ 最終產出： 提供一個單一、完整的 HTML 檔案，其中包含所有需要的 CSS 和 JavaScript。

技術規格：CSS： 必須使用 Tailwind CSS。

JavaScript： 必須使用 原生 JavaScript 處理所有互動邏輯，並使用 Chart.js 來建立數據圖表（例如交通時間與費用的比較圖）。

禁止項目： 禁止使用任何 SVG 圖形或 Mermaid.js。

➤ 網頁架構與內容：

資訊架構： 需設計清晰的資訊架構，例如使用頂部導覽列來切換不同區塊（如：行程總覽、交通住宿、每日行程、美食地圖、行前準備）。

每日行程：

– 需以時間軸 (Timeline) 的方式呈現。

– 需包含點到點的建議路線、交通工具、預估時間。

– 需標示每個景點的建議停留時間。

– 所有景點和路線規劃都必須附上可點擊的 Google Maps 連結。

美食地圖： 需將推薦的餐廳整理成一個獨立區塊，並為每家餐廳附上 Google Maps 連結。可加入篩選器（如：海鮮、小吃、甜點）增加互動性。

➤ 設計與使用者體驗：

視覺風格： 整體設計需簡潔、專業、有質感，並使用淺色系背景。

響應式設計： 網頁必須是響應式的，在電腦、平板和手機上都有良好的瀏覽體驗。

互動性： 除了導覽列切換外，需包含如「美食分類篩選」、「圖表互動提示」等互動元素，提升使用者體驗。
